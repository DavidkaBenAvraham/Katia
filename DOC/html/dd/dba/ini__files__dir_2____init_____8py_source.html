13По умолчанию относительные пути используют текущую директорию, поэтому явно вызывать os.getcwd()
14редко нужно.
15Например, open('file.txt') вызов открывает файл 'file.txt' в текущей директории.
16Если необходимо передать полный путь в виде строки, то можно использовать
17os.path.abspath('file.txt') — getcwd() снова явно не используется.
18
19Path.cwd() из pathlib модуля возвращает путь к текущей директории как
20объект c разными полезными и удобными методами
21такими как .glob('**/*.py').
22
23Директория со скриптом
24Текущая рабочая директория может отличаться от директории с текущим Питон-скриптом. Часто,
25но не всегда можно os.path.dirname(os.path.abspath(__file__)) использовать, чтобы получить директорию
26с текущим Питон скриптом, но это не всегда работает.
27Посмотрите на get_script_dir() функцию, которая поддерживает более общий случай.
28
29Если хочется получить данные из файла, расположенного относительно установленного Питон-модуля,
30то используйте pkgutil.get_data() или setuptools' pkg_resources.resource_string() вместо построения путей c помощью
31__file__. Это работает даже, если ваш пакет упакован в архив. В Python 3.7 появился importlib.resources модуль.
